Docker
Con la tecnología de virtualización, la máquina virtual es quien contempla todo un sistema operativo, herramientas, y aplicación. Un servidor físico que ejecuta tres máquinas virtuales tendría un hipervisor y tres sistemas operativos separados que se ejecutan al mismo tiempo.
Por el contrario, un servidor que ejecuta tres aplicaciones en contenedor con Docker ejecuta un único sistema operativo, y cada contenedor comparte el kernel del sistema operativo con los otros contenedores.
Las partes compartidas del sistema operativo son de solo lectura, mientras que cada contenedor tiene su propio montaje (es decir, una forma de acceder al contenedor) para la escritura. Eso significa que los contenedores son mucho más livianos y usan muchos menos recursos que las máquinas virtuales.
Ventajas de utilizar container vs virtualizacion.
• Despliegue fácil y rápido.
• Portabilidad.
• Ligereza.
• Autosuficiencia.
• Control de versiones de imágenes.
• Ahorro de recursos.
• Ahorro de consumo energético.
• Mayor control.
